General

Playwright is a powerful end-to-end testing framework that enables reliable testing for modern web applications. In our platform, we utilize Playwright's capabilities specifically for API testing, leveraging its robust features for making HTTP requests and validating responses. For more information, visit the official Playwright documentation .

Key Features

  • Cross-platform Support: Works seamlessly across Windows, Linux, and macOS
  • Multiple Language Support: Available for TypeScript, JavaScript, Python, .NET, and Java
  • Auto-waiting: Built-in smart waiting mechanisms that reduce test flakiness
  • Powerful Assertions: Rich set of web-first assertions specifically designed for API testing
  • Request Interception: Ability to intercept, modify, and validate HTTP requests and responses
  • Network Activity Monitoring: Comprehensive tracking of all network requests and responses

API Testing Capabilities

  • Headers Validation: Comprehensive validation of HTTP headers including:
    • Content-Type verification
    • CORS header behavior
    • Cache control directives
    • Connection headers
    • AWS headers validation
  • Response Validation: Detailed validation of API responses including structure, content, and format
  • Performance Metrics: Capture and analyze response times and execution duration

Test Execution Controls

  • Timer Management:
    • Timer starts automatically when test execution begins
    • Timer stops when test execution completes or if there's an error
    • Timer display shows hours:minutes:seconds format (HH:MM:SS)
  • Run Button State:
    • Button is enabled only when at least one endpoint and one test are selected
    • Button state updates automatically when:
      • Individual test or endpoint checkboxes are changed
      • "Select All" checkbox is toggled
      • Page is initially loaded
      • Test execution completes
    • Button is disabled during test execution to prevent multiple runs

Reporter

The Playwright HTML Reporter provides a comprehensive, user-friendly interface for viewing test results. The reporter automatically opens in your default browser after test execution and is also saved locally in the playwright-report folder for future reference.

Example Report

Example of a test run showing Headers Validation test suite with various test cases. The report shows 11 total tests (11 passed, 0 failed, 0 flaky, 1 skipped) with a total execution time of 15.8s. Each test case displays its execution time and specific endpoint being tested.

Reporter Features

  • Test Overview:
    • Total number of tests run
    • Pass/Fail/Flaky/Skipped status indicators with counts
    • Total execution time
    • Timestamp of test execution
  • Detailed Test Results:
    • Individual test execution time
    • Test suite organization and hierarchy
    • Clear pass/fail indicators
    • Descriptive test names and categories
  • Search and Filter:
    • Search functionality for finding specific tests
    • Filter by test status (All, Passed, Failed, Flaky, Skipped)
    • Quick navigation through test suites
  • Test Artifacts:
    • Detailed error messages and stack traces
    • Request/response information
    • Test execution logs
    • Network activity logs

Accessing Reports

  • Automatic Opening: The HTML report automatically opens in your default browser after test execution
  • Local Storage: Reports are saved in the playwright-report folder within your project directory
