PAPI Versions

PAPI version information is always visible in the footer of every page. The version number follows a specific structure that reflects the architectural separation between the framework and test components.

Version Structure

The version number appears in the format: PAPI - F1.5.0/T1.4.0/S1.1.0

  • F1.5.0 - Framework version
    • The 'F' prefix indicates the Framework component
    • Follows semantic versioning (major.minor.patch)
  • T1.4.0 - Tests version
    • The 'T' prefix indicates the Tests component
    • Follows semantic versioning (major.minor.patch)
  • S1.1.0 - SLA version
    • The 'S' prefix indicates the SLA component
    • Follows semantic versioning (major.minor.patch)
    • Represents the Service Level Agreement version

Architectural Separation

PAPI maintains a clear separation between its framework and test components. This separation offers several benefits:

  • Framework and tests can evolve independently
  • Test updates do not impact the framework's functionality
  • Version numbers can progress independently based on their respective changes
  • Easier maintenance and debugging of each component
Note: Always check both version numbers when reporting issues or requesting support, as the behavior might be specific to certain combinations of framework and test versions.

Validation Suite Beta

The Validation Suite is a specialized testing component designed to verify the reliability of the test code itself. Currently in beta, this suite implements scenarios that intentionally trigger test failures to ensure proper error detection and handling.

Purpose

The primary goals of the Validation Suite are:

  • Verify that tests correctly identify and fail when encountering invalid conditions
  • Ensure test assertions work as expected across different scenarios
  • Validate error handling and reporting mechanisms
  • Provide confidence in the test suite's ability to detect actual issues
Beta Status: As this feature is currently in beta, expect ongoing improvements and potential changes to the validation scenarios and implementation.
