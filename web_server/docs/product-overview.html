Project Overview

The API Testing Platform is a comprehensive testing solution designed to validate and verify API endpoints effectively. This platform combines automated testing capabilities with Zephyr Scale for efficient test case management, providing a robust framework for ensuring API reliability and functionality across different environments.

PAPI implements a structured approach to API testing through four main components: Product Selection for easy navigation, Product Console for test execution, Environment Management for different testing contexts, and Zephyr Integration for test case documentation. These components work together to create a unified testing experience.

Product Selection

The Product Selection page (index.html) serves as your starting point, offering:

  • Product Dashboard:
    • Clean, two-column layout displaying available API products
    • Each product card provides:
      • Product name with direct access link
      • Base URL in monospace format
      • Total number of testable endpoints
  • Quick Navigation:
    • Direct links to product testing consoles
    • Easy access to product documentation

Product Console

The Product Console (product.html) is your testing command center, featuring:

  • Testing Matrix:
    • Comprehensive endpoint selection panel
    • Test suite configuration options
    • Batch testing capabilities
  • Real-time Monitoring:
    • Live test execution status
    • Execution time tracking
    • Immediate test results display
  • Test Management:
    • Flexible endpoint selection with group actions
    • Customizable test suite combinations
    • Detailed test execution reports

Environments

The Environments page (environments.html) enables environment management:

  • Environment Control:
    • Switch between development environments
    • View and verify active BASE_URL
    • Environment-specific configuration management
  • Safety Features:
    • Development-only environment restrictions
    • Confirmation dialogs for environment changes
    • Clear visibility of active environment
Note: When selecting a new environment, the change will be applied globally across the entire project. This means all subsequent API calls and tests will use the newly selected environment's BASE_URL. Always verify your environment selection before running tests to ensure you're working in the intended environment.

Zephyr Integration

The Zephyr Integration (zephyr.html) provides test case management:

  • Test Case Management:
    • Automated test case generation
    • CSV export in Zephyr-compatible format
    • Seamless integration with Zephyr Scale
  • Documentation Features:
    • Detailed test step documentation
    • Test data and expected results tracking
    • Standardized test case format for consistency
