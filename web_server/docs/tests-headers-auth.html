Headers Suite Documentation (Authentication Annotated)

Summary

This page documents all tests in headers.spec.js and indicates for each test whether it is related to Authentication or not. Use this as a reference for security and compliance reviews.

Test Cases Overview

  1. Response Has Correct Content-Type Not Authentication

    Verifies all responses include the Content-Type: application/json header.

  2. CORS No Origin Returns Wildcard Not Authentication

    Verifies CORS wildcard behavior when no Origin header is sent.

  3. CORS Origin Echoed Back Not Authentication

    Verifies CORS behavior when a specific Origin header is provided.

  4. Cache Control Directives Not Authentication

    Ensures the response includes all required cache-control directives.

  5. Verify Connection header is keep-alive Not Authentication

    Ensures the connection header is set to keep-alive.

  6. Required AWS Headers Present Not Authentication

    Ensures required AWS API Gateway headers are present in the response.

  7. Missing Headers Validation Authentication

    Verifies the server returns 401 when any required authentication header (Glasseson-Client-Id, Glasseson-Profile-Name, Glasseson-Native-App-Id) is missing.

  8. All Required Headers Missing Authentication

    Verifies the server returns 401 when all required authentication headers are missing.

  9. Empty Headers Validation Authentication

    Verifies the server returns 401 when required authentication headers are present but empty.

  10. All Required Headers Empty Authentication

    Verifies the server returns 401 when all required authentication headers are empty.

  11. Invalid Header Values Authentication

    Verifies the server returns 401 when required authentication headers have invalid values.

  12. All Invalid Header Values Authentication

    Verifies the server returns 401 when all required authentication headers have invalid values.

  13. Missing Glasseson-Session-Id Header Authentication

    Verifies the server returns 500 (or 401 for some endpoints) when the session ID header is missing.

Legend

  • Authentication — Test is directly related to authentication/session enforcement.
  • Not Authentication — Test is not related to authentication.
